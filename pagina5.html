ACERCA DE LOS REPOSITORIOS REMOTOS CON GITHUB

El enfoque colaborador de GitHub para el desarrollo depende en publicar confirmaciones de cambios desde tu repositorio local para que otras personas lo vean, lo extraigan y lo actualicen. Una URL remota es la manera de Git de decir "el lugar donde se almacena tu código". Esa URL podría ser tu repositorio en GitHub o la bifurcación de otro usuario o incluso en un servidor completamente diferente.

SOLO PUEDES SUBIR A DOS TIPOS DE DIRECCIONES URL:
1)Una URL HTTPS como https://github.com/user/repo.git
2)Una URL SSH como git@github.com:user/repo.git
Git asocia una URL remota con un nombre y tu remoto predeterminado generalmente se llama origen
